# 03. IP Address Plan

## IP 주소 설계 목적

본 문서는 RE:NET Enterprise Network Lab에서 사용할 IP 주소 체계를 정리한 문서이다.

네트워크를 부서별, 역할별로 분리하기 위해 각 VLAN마다 서로 다른 IP 대역을 할당한다.

이를 통해 네트워크 관리 편의성을 높이고, 보안 정책을 명확하게 적용할 수 있도록 한다.

---

## IP 주소 설계 원칙

본 프로젝트에서는 사설 IP 대역인 `10.10.0.0/16`을 사용한다.

각 VLAN은 `/24` 단위로 분리한다.

예를 들어 개발팀은 `10.10.10.0/24`, 인사팀은 `10.10.20.0/24`처럼 구성한다.

이렇게 설계하면 VLAN 번호와 IP 대역을 직관적으로 연결할 수 있다.

| VLAN ID | 네트워크 대역 |
|---:|---|
| VLAN 10 | 10.10.10.0/24 |
| VLAN 20 | 10.10.20.0/24 |
| VLAN 30 | 10.10.30.0/24 |
| VLAN 40 | 10.10.40.0/24 |
| VLAN 99 | 10.10.99.0/24 |

---

| VLAN ID | 이름 | 용도 | 네트워크 대역 | Gateway |

## DHCP 할당 범위

사용자 단말이 위치하는 DEV, HR, GUEST VLAN은 DHCP를 통해 IP 주소를 자동으로 할당받는다.

서버존과 관리망은 안정적인 운영을 위해 Static IP를 사용한다.

| VLAN ID | 이름 | DHCP 범위 |
|---:|---|---|
| 10 | DEV | 10.10.10.100 ~ 10.10.10.200 |
| 20 | HR | 10.10.20.100 ~ 10.10.20.200 |
| 30 | GUEST | 10.10.30.100 ~ 10.10.30.200 |
| 40 | SERVER | Static IP 사용 |
| 99 | MGMT | Static IP 사용 |

서버는 서비스 안정성을 위해 DHCP가 아닌 Static IP를 사용한다.

## 라우터 간 연결 IP 계획

라우터 간 연결 구간은 `/30` 대역을 사용한다.

`/30`은 네트워크 주소 1개, 브로드캐스트 주소 1개, 사용 가능한 IP 2개를 제공하므로 라우터 간 Point-to-Point 연결에 적합하다.

| 연결 구간 | 네트워크 대역 | 장비 A IP | 장비 B IP |
|---|---|---|---|
| edge-r1 ↔ fw-01 | 10.10.254.0/30 | 10.10.254.1 | 10.10.254.2 |
| fw-01 ↔ core-r1 | 10.10.254.4/30 | 10.10.254.5 | 10.10.254.6 |
| fw-01 ↔ core-r2 | 10.10.254.8/30 | 10.10.254.9 | 10.10.254.10 |
| core-r1 ↔ core-r2 | 10.10.254.12/30 | 10.10.254.13 | 10.10.254.14 |
| core-r1 ↔ l3-sw01 | 10.10.254.16/30 | 10.10.254.17 | 10.10.254.18 |
| core-r2 ↔ l3-sw01 | 10.10.254.20/30 | 10.10.254.21 | 10.10.254.22 |

## ISP 연결 IP 계획

BGP 실습을 위해 가상의 ISP-A, ISP-B를 구성할 예정이다.

외부 연결 구간은 실습용 문서 IP 대역을 사용한다.

| 연결 구간 | 네트워크 대역 | 회사 측 IP | ISP 측 IP |
|---|---|---|---|
| edge-r1 ↔ ISP-A | 203.0.113.0/30 | 203.0.113.2 | 203.0.113.1 |
| edge-r1 ↔ ISP-B | 198.51.100.0/30 | 198.51.100.2 | 198.51.100.1 |

## IP 주소 설계 이유

### 1. VLAN 번호와 IP 대역 일치

VLAN 10은 `10.10.10.0/24`, VLAN 20은 `10.10.20.0/24`처럼 설계하였다.

이렇게 하면 VLAN 번호만 보고도 해당 네트워크 대역을 쉽게 예측할 수 있다.

---

### 2. 사용자망과 서버망 분리

사용자 단말이 위치하는 DEV, HR, GUEST VLAN과 서버가 위치하는 SERVER VLAN을 분리하였다.

이를 통해 사용자망에서 서버존으로 접근할 때 필요한 서비스만 허용할 수 있다.

---

### 3. 관리망 별도 구성

네트워크 장비의 관리 IP는 MGMT VLAN에 따로 배치하였다.

일반 사용자망에서 라우터, 스위치, 방화벽에 직접 접근하지 못하게 하여 보안을 강화한다.

---

### 4. 서버와 장비는 Static IP 사용

서버와 네트워크 장비는 주소가 자주 바뀌면 운영에 문제가 생길 수 있다.

따라서 DHCP가 아닌 Static IP를 사용한다.

---

### 5. 라우터 간 연결은 /30 사용

라우터와 라우터 사이의 연결은 두 개의 IP만 필요하다.

따라서 IP 주소 낭비를 줄이기 위해 `/30` 대역을 사용한다.

---

## 정리

본 IP 주소 계획은 VLAN, 서버존, 관리망, 라우터 간 연결, ISP 연결을 체계적으로 구분하기 위해 작성하였다.

향후 실제 구성 단계에서는 이 문서를 기준으로 각 장비의 인터페이스 IP, Gateway, DHCP 범위, 라우팅 설정을 구성한다.
