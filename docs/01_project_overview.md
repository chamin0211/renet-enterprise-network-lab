# 01. Project Overview

## 프로젝트명

RE:NET Enterprise Network Lab

---

## 프로젝트 목적

본 프로젝트는 네트워크 엔지니어 취업 포트폴리오를 목적으로, 가상의 기업 네트워크를 직접 설계하고 구축하는 것을 목표로 한다.

단순히 장비를 연결하는 수준이 아니라, 실제 기업 네트워크 운영에서 필요한 VLAN 분리, 라우팅, 방화벽 정책, NAT, VPN, 모니터링, 자동화, 장애 대응 과정을 포함한다.

이 프로젝트를 통해 네트워크 엔지니어 직무에서 요구되는 설계 능력, 구축 능력, 운영 능력, 트러블슈팅 능력을 단계적으로 학습한다.

---

## 프로젝트 배경

네트워크 엔지니어는 기업 내부망 구성, 라우터 및 스위치 운영, 방화벽 정책 관리, 장애 대응, 네트워크 모니터링 등의 업무를 수행한다.

따라서 본 프로젝트에서는 약 100명 규모의 중소기업 환경을 가정하고, 실제 기업 네트워크에서 자주 사용되는 구조를 단순화하여 구현한다.

가상의 회사인 RE:NET Corp는 개발팀, 인사/관리팀, 게스트 사용자, 서버존, 네트워크 관리망으로 구성된다.

각 영역은 VLAN으로 분리되며, 필요한 통신만 허용하고 불필요한 접근은 차단하는 구조로 설계한다.

---

## 주요 요구사항

본 프로젝트의 주요 요구사항은 다음과 같다.

- 부서별 네트워크 분리
- 게스트망과 내부망 분리
- 서버존 보호
- 관리망 접근 제한
- 내부 라우팅 구성
- 외부 인터넷 연결 구성
- 원격 관리자 VPN 접속 구성
- 네트워크 상태 모니터링
- 장애 발생 시 원인 분석 및 복구
- 반복적인 설정 작업 자동화

---

## 최종 구현 범위

본 프로젝트에서 최종적으로 구현할 기능은 다음과 같다.

| 구분 | 구현 내용 |
|---|---|
| 네트워크 설계 | 토폴로지, IP 주소 계획, VLAN 설계 |
| 스위칭 | VLAN, Trunk, Access Port 구성 |
| 라우팅 | Inter-VLAN Routing, OSPF, BGP |
| 보안 | Firewall Policy, NAT, 게스트망 격리 |
| 원격 접속 | VPN을 통한 관리자 접속 |
| 서비스 | DHCP, DNS, Web Server 구성 |
| 모니터링 | Prometheus/Grafana 기반 상태 시각화 |
| 자동화 | Ansible 기반 설정 백업 및 점검 |
| 장애 대응 | 장애 시나리오 작성, 원인 분석, 복구 문서화 |

---

## 프로젝트 진행 방식

프로젝트는 다음 단계로 진행한다.

1. 네트워크 설계 문서 작성
2. 실습 환경 구축
3. VLAN 및 IP 주소 구성
4. Inter-VLAN Routing 구성
5. OSPF 내부 라우팅 구성
6. BGP ISP 이중화 구성
7. NAT 및 Firewall 정책 구성
8. VPN 원격 접속 구성
9. DHCP/DNS/Web Server 구성
10. Prometheus/Grafana 모니터링 구성
11. Ansible 자동화 구성
12. 장애 대응 시나리오 작성
13. GitHub README 및 포트폴리오 정리

---

## 기대 효과

이 프로젝트를 통해 다음과 같은 역량을 기를 수 있다.

- 네트워크 구조를 직접 설계하는 능력
- IP 주소와 VLAN을 체계적으로 나누는 능력
- 라우팅 테이블과 경로 선택 과정을 이해하는 능력
- 방화벽 정책을 설계하고 적용하는 능력
- 장애 상황에서 원인을 찾고 복구하는 능력
- 네트워크 상태를 모니터링하는 능력
- 반복 작업을 자동화하는 능력
- 구축 과정과 학습 내용을 문서화하는 능력

---

## 포트폴리오 활용 방향

이 프로젝트는 단순한 실습 결과물이 아니라, 네트워크 엔지니어 지원 시 사용할 수 있는 포트폴리오로 구성한다.

GitHub에는 설계 문서, 설정 파일, 자동화 스크립트, 모니터링 구성, 장애 대응 문서를 함께 정리한다.

면접에서는 단순히 “구현했다”가 아니라, 왜 이런 구조로 설계했는지, 어떤 문제가 발생했고 어떻게 해결했는지, 어떤 네트워크 개념을 학습했는지를 설명할 수 있도록 준비한다.
